## tctl auth sign

When invoked with a `--format=db` flag, produces a CA certificate, a client certificate and a private key file used for configuring Database Access with self-hosted database instances.

```bash
tctl auth sign --format=db --host=db.example.com --out=server --ttl=2190h
```

<table>
  <thead>
    <tr>
      <td>Flag</td>
      <td>Description</td>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`--format`</td>
      <td>When given value `db`, produces secrets in database compatible format.</td>
    </tr>
    <tr>
      <td>`--host`</td>
      <td>Server name to encode in the certificate. Must match the hostname Teleport will be connecting to the database at.</td>
    </tr>
    <tr>
      <td>`--out`</td>
      <td>Name prefix for output files.</td>
    </tr>
    <tr>
      <td>`--ttl`</td>
      <td>Certificate validity period.</td>
    </tr>
  </tbody>
</table>

## tctl db ls

Administrative command to list all databases registered with the cluster.

```bash
tctl db ls
tctl db ls --format=yaml
```

<table>
  <thead>
    <tr>
      <td>Flag</td>
      <td>Description</td>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`--format`</td>
      <td>Output format, one of `text`, `yaml` or `json`. Defaults to `text`.</td>
    </tr>
  </tbody>
</table>
